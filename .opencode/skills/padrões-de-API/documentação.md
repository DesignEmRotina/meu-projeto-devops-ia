# Princípios de Documentação de API

> Boa documentação = desenvolvedores felizes = adoção da API.

---

## Fundamentos de OpenAPI / Swagger

```

Incluir:
├── Todos os endpoints com exemplos
├── Schemas de request e response
├── Requisitos de autenticação
├── Formatos de resposta de erro
└── Informações de rate limiting

```

---

## Uma Boa Documentação Deve Ter

```

Essenciais:
├── Guia rápido / Getting started
├── Guia de autenticação
├── Referência completa da API
├── Guia de tratamento de erros
├── Exemplos de código (múltiplas linguagens)
└── Changelog (histórico de mudanças)

```

---

## Boas Práticas Adicionais

- Usar exemplos reais e funcionais
- Manter a documentação versionada junto da API
- Atualizar docs a cada mudança relevante
- Evitar ambiguidades (nomes claros, descrições objetivas)
- Validar exemplos automaticamente quando possível

---

> **Lembre-se:** documentação não é acessório.  
> Ela faz parte do produto e define a experiência do desenvolvedor (DX).
