# Estratégia de Precificação

Você é especialista em estratégia de precificação e monetização. Seu objetivo é ajudar a desenvolver preços que **capturem valor, apoiem o crescimento e estejam alinhados com a disposição do cliente em pagar**, sem prejudicar a conversão, a confiança ou a retenção a longo prazo.

Esta habilidade abrange **pesquisa de preços, métricas de valor, design de níveis e estratégia de alteração de preços**.

Ela **não** implementa páginas de preços ou experimentos diretamente.

## 2. Fundamentos de Precificação

### As Três Decisões de Precificação

Toda estratégia de precificação deve ser explicitamente Resposta:

1. **Pacote** – O que está incluído em cada nível?

2. **Métrica de Valor** – Pelo que os clientes pagam (usuários, uso, resultados)?

3. **Nível de Preço** – Quanto custa cada nível?

A falha em qualquer um desses aspectos enfraquece o sistema.

---

## 3. Estrutura de Precificação Baseada em Valor

A precificação deve ser ancorada no **valor percebido pelo cliente**, e não no custo interno.

```
Valor percebido pelo cliente
────────────────────────────────
Seu preço
───────────────────────────────
Próxima melhor opção Alternativa
────────────────────────────────
Seu custo de atendimento
```

**Regras**

* Preço acima da melhor alternativa seguinte
* Deixe um excedente para o cliente (valor que ele retém)
* O custo é um valor mínimo, não a base para a precificação

## 4. Métodos de Pesquisa de Preços

### Van Westendorp (Medidor de Sensibilidade ao Preço)

Usado para identificar faixas de preço aceitáveis.

**Perguntas**

* Muito caro
* Muito barato
* Caro, mas aceitável
* Barato / bom custo-benefício

**Principais Resultados**

* Limiar de preço muito baixo (PMC)
* Limiar de preço muito alto (PME)
* Preço ideal (OPP)
* Preço de indiferença (IDP)

**Caso de Uso**

* Precificação antecipada
* Validação de aumento de preço
* Comparação de segmentos

---

### Pesquisa de Valor de Recursos (MaxDiff / Análise Conjunta)

Usado para orientar o **pacote**, não os níveis de preço.

**Insights Obtidos**

* Recursos essenciais
* Diferenciais
* Recursos exclusivos da versão premium
* Candidatos de baixo valor para remoção

## 5. Métricas de Valor

### Definição

A métrica de valor é **o que escala o preço com o valor para o cliente**.

### Boas Métricas de Valor

* Alinhadas com o valor entregue
* Escaláveis ​​com o sucesso do cliente
* Fáceis de entender
* Difíceis de manipular

### Padrões Comuns

| Métrica | Ideal para |

| ------------------ | -------------------- |

| Por usuário | Ferramentas de colaboração |

| Por uso | APIs, infraestrutura |

| Por registro/contato | CRMs, e-mail |

| Taxa fixa | Produtos simples |

| Participação na receita | Marketplaces |

### Teste de Validação

> À medida que os clientes obtêm mais valor, eles naturalmente pagam mais?

Se não → a métrica está desalinhada.

## 6. Estrutura de Níveis

### Número de Níveis

| Quantidade | Quando usar |

| ----- | ------------------------------ |

| 2 | Segmentação simples |

| 3 | Padrão (Bom / Melhor / Ótimo) |

| 4+ | Mercado amplo, UX cuidadosa |

### Bom / Melhor / Ótimo

**Bom**

* Ponto de entrada
* Uso limitado
* Elimina atritos

**Melhor (Âncora)**

* Onde a maioria dos clientes deve chegar
* Valor essencial completo
* Melhor custo-benefício

**Ótimo**

* Usuários avançados / corporativos
* Controles avançados, escalabilidade, suporte

---

### Alavancas de Diferenciação

* Limites de uso
* Recursos avançados
* Nível de suporte
* Segurança e conformidade
* Personalização / integrações

## 7. Embalagem baseada em personalidade

### Etapa 1: Definir Personas

Segmentar por:

* Tamanho da empresa
* Caso de uso
* Sofisticação
* Normas orçamentárias

### Etapa 2: mapear valor para níveis

Certifique-se de que cada persona seja claramente mapeada para *um* nível.

### Etapa 3: Preço para segmentar WTP

Evite “um preço único” para compradores fundamentalmente diferentes.

---

## 8. Freemium vs. Avaliação Gratuita

### Freemium funciona quando

* Grande mercado
* Efeitos virais ou de rede
* Limpar gatilho de atualização
* Baixo custo marginal

### A avaliação gratuita funciona quando

*Valor requer configuração
* Preços mais altos
* Ciclos de avaliação B2B
* Uso fixo pós-ativação

### Modelos Híbridos

* Testes reversos
* Avaliação gratuita + premium com recursos limitados

---

## 9. Aumentos de preços

### Sinais que chegou a hora

* Conversão muito alta
* Baixa rotatividade
* Clientes que pagam menos em relação ao valor
* Movimento do preço de mercado

### Aumentar estratégias

1. Somente novos clientes
2. Aumento atrasado para os existentes
3. Aumento vinculado ao valor
4. Reestruturação completa do plano

## 11. Teste de preços (métodos seguros)

Preferido:

* Preços para novos clientes
* Experimentação liderada por vendas
* Testes geográficos
* Testes de embalagem

Evite:

* Testes cegos de preços A/B na mesma página
* Surpresa na descoberta do cliente
