# Padrões de Teste E2E

Crie conjuntos de testes de ponta a ponta confiáveis, rápidos e de fácil manutenção que ofereçam confiança para entregar código rapidamente e detectar regressões antes que os usuários as encontrem.

## Use esta habilidade quando:

- Implementar automação de testes de ponta a ponta
- Depurar testes instáveis ​​ou não confiáveis
- Testar fluxos de trabalho críticos do usuário
- Configurar pipelines de teste de CI/CD
- Testar em vários navegadores
- Validar requisitos de acessibilidade
- Testar designs responsivos
- Estabelecer padrões de teste de ponta a ponta

## Não use esta habilidade quando:

- Você precisar apenas de testes unitários ou de integração
- O ambiente não suportar automação de interface do usuário estável
- Você não puder provisionar contas ou dados de teste seguros

## Instruções

1. Identifique as jornadas críticas do usuário e os critérios de sucesso.

2. Crie seletores estáveis ​​e estratégias de dados de teste.

3. Implemente testes com novas tentativas, rastreamento e isolamento.

4. Execute em CI com paralelização e captura de artefatos.

## Segurança

- Evite executar testes destrutivos em produção.

- Use dados de teste dedicados e remova informações confidenciais da saída.
