# Geração de Especificações OpenAPI

Padrões abrangentes para criar, manter e validar especificações OpenAPI 3.1 para APIs RESTful.

## Use esta habilidade quando:

- Criar documentação de API do zero
- Gerar especificações OpenAPI a partir de código existente
- Projetar contratos de API (abordagem de design em primeiro lugar)
- Validar implementações de API em relação às especificações
- Gerar SDKs de cliente a partir das especificações
- Configurar portais de documentação de API

## Não use esta habilidade quando:

- A tarefa não estiver relacionada à geração de especificações OpenAPI
- Você precisar de um domínio ou ferramenta diferente, fora deste escopo

## Instruções

- Esclareça os objetivos, restrições e entradas necessárias.

- Aplique as melhores práticas relevantes e valide os resultados.

- Forneça etapas práticas e verificação.

- Se forem necessários exemplos detalhados, abra `recursos/manual-de-implementação.md`.
